# Code Review Required

## Rule
All code changes require review before merge. Review is performed by the `dev_implement-review` hat using `gh pr review` (approve/request-changes). Code review happens on the PR, not via issue comments.

## Applies To
All code-producing hats (`dev_implement-red`, `dev_implement-green`, `dev_implement-refactor`). Applies to all project repo contributions.

## Verification
Every code change passes through the `dev_implement-review` hat before proceeding to QE verification at `eng:qe:verify`. The PR review record confirms the review occurred. No code reaches `eng:qe:verify` without an approved PR review.
