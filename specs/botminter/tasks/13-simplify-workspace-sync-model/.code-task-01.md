### Migration strategy

During `sync_workspace()`, after the submodule update step, check if the team submodule is on a branch other than `main`. If so:
1. `git checkout main` in the submodule. If `main` doesn't exist locally, try `git checkout -b main origin/main`. If that also fails (no remote tracking), the submodule is likely on a detached HEAD or unusual state â€” log a warning and continue without migrating.
2. Emit `SyncEvent::BranchMigrated { submodule, from }` â€” emit **unconditionally** (not gated on `verbose`) since this is a one-time significant operational change.
3. Delete the old member branch locally: `git branch -d <old-branch>`. Use `-d` (safe delete), not `-D`. If `-d` refuses because the branch has unmerged commits, log a warning but do not force-delete â€” the operator can investigate.

Apply the same migration to each project submodule.

### Error handling fix

Fix **both** `.ok()` sites in `sync_workspace()`:

**Line 106 (team submodule update)** and **line 126 (project submodule update):**
```rust
// Before (both sites):
git_cmd(ws_root, &[...]).ok(); // silently swallows errors

// After:
if let Err(e) = git_cmd(ws_root, &[...]) {
    eprintln!("Warning: failed to update submodule: {:#}", e);
}
```

Design principle: `SyncEvent` is for normal operational events the operator should know about (migration happened, project provisioned). `eprintln!` is for warnings about failures that don't block the operation. Submodule update failures are warnings â€” sync should continue even if one submodule can't be updated.

## Acceptance Criteria

- AC-01: Given a workspace with team submodule, when `sync_workspace()` runs, then team submodule is on `main` (not a member branch). After sync, `git rev-parse --abbrev-ref HEAD` in team/ returns `main`.
- AC-02: `checkout_member_branch()` function no longer exists in `sync.rs`. Dead `SyncEvent` variants (`BranchAlreadyOnIt`, `BranchCheckedOut`, `BranchCreated`) are also removed.
- AC-06: When submodule update fails at **either** the team submodule (line 106) or project submodule (line 126) sites, the error is logged to stderr with an actionable message (not silently swallowed via `.ok()`).
- AC-07: Given an existing workspace where team submodule is on member branch `engineer-bob`, when `sync_workspace()` runs, then submodule is switched to `main` and old branch is deleted (if cleanly merged). A `SyncEvent::BranchMigrated` is emitted unconditionally (visible without `-v` flag). If the old branch has unmerged commits, a warning is logged and the branch is preserved.
- AC-08: `just test` passes (unit + e2e), `just exploratory-test` passes after all changes. E2E sync tests should verify submodules are on `main`. No regressions in existing sync tests.

## Complexity
