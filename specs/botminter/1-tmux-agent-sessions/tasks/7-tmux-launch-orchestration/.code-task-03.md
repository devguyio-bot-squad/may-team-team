# Task 03: Add single-start orchestration — skip-if-live, dead-window cleanup

## Context
- **Story**: #7 — STORY-03: Wire tmux into launch functions and start/stop orchestration

- **Acceptance Criteria**: AC-01 (partial), AC-07a, AC-07b, AC-08, AC-18
- **Design Doc**: team/specs/botminter/1-tmux-agent-sessions/design.md

## Objective
Implement the single-member start path in `start_local_members()`: create the tmux session if it doesn't exist, skip members with live windows, and clean up dead windows before re-creating them. After this task, `bm start bob` works correctly in all scenarios — fresh start, adding to existing session, restarting a stopped member, and skipping an already-running member.

## Background
The full-start path (Task 02) always destroys and recreates the session. The single-member path (`bm start bob`) is more nuanced:

1. **No session exists** → create session, create window for bob
2. **Session exists, bob not running** → add window to existing session
3. **Session exists, bob has live window** → skip with warning (don't kill running process)
4. **Session exists, bob has dead window** → remove dead window, create fresh one

This logic uses the STORY-02 window query methods (`window_exists()`, `is_pane_dead()`, `pane_pid()`) and removal methods (`remove_dead_window()`).

## Technical Requirements
1. For single start (`member_filter.is_some()`): `if !tmux.exists() { tmux.create()? }`
2. Before each member launch in the loop, add skip-if-live check:
   ```
   if tmux.window_exists(&member) && !tmux.is_pane_dead(&member)? {
       let running_pid = tmux.pane_pid(&member)?;
       warn!("Member '{}' already running (pid {}). Use 'bm stop {}' first, or inspect with: tmux -L botminter attach -t bm-{}:{}", member, running_pid, member, team_name, member);
       continue;
   }
   ```
3. Before each member launch, add dead-window cleanup:
   ```
   if tmux.window_exists(&member) && tmux.is_pane_dead(&member)? {
       tmux.remove_dead_window(&member)?;
   }
   ```
4. The skip-if-live and dead-window checks go inside the member loop (which runs for both paths). On full-start the session was just destroyed and recreated (Task 02), so `window_exists()` will always return false — the guards are defensive no-ops for full-start but functionally meaningful for single-start.
5. Warning message for skip-if-live MUST include the member name and running PID for debuggability
6. Use unique session names in integration tests to avoid collisions with parallel `cargo test`
7. **Known TOCTOU race:** Two concurrent `bm start bob` invocations (e.g., CLI + daemon HTTP handler) can race between `window_exists()` and `create_window()`. The failure mode is a tmux "duplicate window name" error, which is safe (no data corruption). Add a note in the code acknowledging this race and documenting that the error is non-destructive. Add an integration test that exercises concurrent invocations to verify the failure mode produces a clear error, not silent corruption.

## Implementation Approach
1. Add the single-start session creation branch to `start_local_members()`:
   - After the existing full-start logic from Task 02, add the `else` branch for `member_filter.is_some()`
2. Add the per-member guards before the launch call in the member loop:
   - Skip-if-live: check window exists and pane is alive → skip with warning, push to `result.skipped`
   - Dead-window cleanup: check window exists and pane is dead → call `remove_dead_window()`
3. Write integration tests:
   - **Fresh single start**: no session → `bm start bob` → session created with one window `bob`
   - **Additive single start**: session exists with `bob` → `bm start cos` → `cos` window added, `bob` undisturbed
   - **Skip-if-live**: `bob` running → `bm start bob` → skip with warning, bob's process undisturbed
   - **Dead-window restart**: stop bob → bob window dead → `bm start bob` → dead window removed, fresh window created
   - **TMUX_TMPDIR unset**: verify socket under `/tmp/tmux-<uid>/` even when `TMUX_TMPDIR` is set

## Acceptance Criteria

1. **Fresh Single Start — Session Creation**
   - Given no tmux session exists
   - When `bm start bob` is called
   - Then a session `bm-<team>` is created with exactly one window named `bob`

2. **Additive Single Start — Window Added**
   - Given session `bm-<team>` exists with window `bob`
   - When `bm start cos` is called
   - Then window `cos` is added to the existing session, and window `bob` remains undisturbed with its original process running

3. **Skip-If-Live — Warning**
   - Given `bob` is running in a live tmux window
   - When `bm start bob` is called
   - Then bob is skipped with a warning including the running PID, and the existing process is not affected

4. **Dead-Window Restart**
   - Given window `bob` has a dead pane (process exited)
   - When `bm start bob` is called
   - Then the dead window is removed and a new window `bob` is created with a live process

5. **Stop Then Start Cycle**
   - Given agents are running → `bm stop` → windows show dead panes
   - When `bm start bob` is called
   - Then the dead window is removed and a fresh window with a live process is created
