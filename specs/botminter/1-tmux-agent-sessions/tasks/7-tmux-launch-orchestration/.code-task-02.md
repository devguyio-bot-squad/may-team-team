# Task 02: Wire tmux session lifecycle into `start_local_members()` — full start path

## Objective
Add tmux session management to the top of `start_local_members()`: prerequisite checks, session construction, and the full-start path (destroy existing session + create fresh one). Wire the `&TmuxSession` handle through to the launch functions modified in Task 01. After this task, `bm start` (all members) creates a tmux session with one window per agent.

## Background
`start_local_members()` is the orchestration entry point for launching members. Currently it checks for `ralph` in PATH, resolves credentials, iterates members, and spawns processes. This task adds tmux session management at the top — prerequisite checks (`check_tmux_available()`, `TmuxConfig::ensure_written()`), session lifecycle, and threading the `TmuxSession` handle through to the launch calls.

The full-start path (`member_filter.is_none()`) destroys any existing session and creates a fresh one. This ensures a clean slate — old dead panes, stale scrollback, and previous session state are all cleared.

The daemon path also goes through `start_local_members()` (via the daemon's HTTP handler), so tmux integration happens transparently. The daemon inherits `PATH` (including tmux) from the operator's environment at startup and can access the tmux socket since it runs on the same host as the same user.

## Technical Requirements
1. At the top of `start_local_members()`, add: `TmuxSession::check_tmux_available()?` and `TmuxConfig::ensure_written()?`
2. Construct `let tmux = TmuxSession::new(&team.name)?`
3. For full start (`member_filter.is_none()`): call `tmux.destroy_if_exists()?` then `tmux.create()?`. Log "Destroying previous session bm-<team>" when an existing session is found. After session creation, print the attach command to stderr: `"tmux session bm-<team> created — attach with: tmux -L botminter attach -t bm-<team>"`.
4. Pass `&tmux` to `launch_ralph()` and `launch_brain()` in the member loop
5. The existing 2-second sleep verification after launch (`start_members.rs:219`) MUST be preserved as a post-launch health check. `create_window()` catches immediate exits (binary-not-found), but a member that crashes at 500ms would be reported as "launched" without this check. Keep the sleep + `is_alive()` gate: if the PID dies within 2 seconds, remove from state.json and push to `errors`.
6. No changes to stop behavior — existing PID-based `kill(pid, SIGTERM)` works because the PID from `create_window()` is the actual process inside the pane

## Implementation Approach
1. Add tmux imports at top of `start_members.rs`
2. After the `ralph` prerequisite check (line 80), add:
   ```rust
   TmuxSession::check_tmux_available()?;
   TmuxConfig::ensure_written()?;
   let tmux = TmuxSession::new(&team.name)?;
   ```
3. Before the member loop (after credential resolution), add session lifecycle:
   ```rust
   if member_filter.is_none() {
       if tmux.exists() {
           eprintln!("Destroying previous session bm-{}", team.name);
       }
       tmux.destroy_if_exists()?;
       tmux.create()?;
   }
   ```
4. In the member loop, update launch calls to pass `&tmux`
5. Consider removing or simplifying the 2-second sleep (line 219) since `create_window()` PID validation catches immediate exits
6. Write integration tests:
   - Full start with 3 members → `tmux list-windows` shows all 3 with correct names
   - Session uses `botminter` socket (`-L botminter`)
   - Each window named after its member
   - Stop → windows remain with dead panes
   - Second full start → old session destroyed, fresh session created
   - Daemon-triggered launch: verify member runs inside tmux window

## Acceptance Criteria

1. **Full Start Creates Session with Windows**
   - Given a team with 3 members
   - When `bm start` (full start) is called
   - Then a tmux session named `bm-<team>` exists on the `botminter` socket, with exactly 3 windows, each named after its member

2. **Previous Session Destroyed on Full Start**
   - Given a tmux session `bm-<team>` exists from a previous run
   - When `bm start` (full start) is called
   - Then the old session is destroyed and a new session is created with fresh windows

3. **Daemon-Triggered Launch Uses Tmux**
   - Given the daemon is running
   - When a member start is triggered via the daemon's HTTP API
   - Then the member runs inside a tmux window in the `bm-<team>` session

4. **Stop Preserves Windows with Scrollback**
   - Given agents are running in tmux
   - When `bm stop` is called
   - Then agent processes are terminated but tmux windows remain with dead panes
   - And `tmux capture-pane -t bm-<team>:<member> -p` returns non-empty output containing text from the agent's execution
