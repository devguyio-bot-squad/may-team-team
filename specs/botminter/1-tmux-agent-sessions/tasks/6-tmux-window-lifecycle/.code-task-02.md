# Task 02: Implement window query methods — `window_exists()`, `is_pane_dead()`, `pane_pid()`, `list_windows()`

- **Requirements**: SESS-02, LIFE-03
- **Acceptance Criteria**: AC-08 (partial — dead pane detection verified via integration test, not via `bm stop` E2E)
- **Design Doc**: team/specs/botminter/1-tmux-agent-sessions/design.md

## Objective
Add window state query methods to `TmuxSession`: check if a window exists, detect dead panes (process exited but window retained by `remain-on-exit on`), retrieve the process PID of a window, and list all windows with their state. These methods are used by STORY-03 for orchestration logic (skip live windows, clean up dead windows).

## Background
With `remain-on-exit on` in the tmux config, windows stay alive after their process exits — the pane shows "Pane is dead (status N)". This is intentional for post-mortem inspection. The orchestration layer needs to distinguish live windows (process running) from dead windows (process exited, pane retained) to handle restart scenarios correctly: skip members with live windows, remove dead windows before re-creating.

## Technical Requirements
1. Implement `pub fn window_exists(&self, name: &str) -> bool` — uses `tmux list-windows -F '#{window_name}'` and checks if the name appears in output
2. Implement `pub fn is_pane_dead(&self, window: &str) -> Result<bool>` — uses `tmux display-message -t <session>:<window> -p '#{pane_dead}'`, returns true if output is `"1"`
3. Implement `pub fn pane_pid(&self, window: &str) -> Result<u32>` — uses `tmux display-message -t <session>:<window> -p '#{pane_pid}'`, parses output to `u32`
4. Implement `pub fn list_windows(&self) -> Result<Vec<TmuxWindow>>` — uses `tmux list-windows -F` with a delimiter-separated format string to extract index, name, pane_pid, and pane_dead for each window; parses into `Vec<TmuxWindow>`
5. All methods MUST use the `tmux_cmd()` helper and `-L <socket>` for socket isolation
6. Error messages MUST include session name and window name for debuggability

- Task 01 must be complete (`create_window()` — needed for integration tests)

## Implementation Approach
1. `window_exists()`: run `tmux_cmd().args(["-L", socket, "list-windows", "-t", session, "-F", "#{window_name}"])`, collect stdout lines, check if `name` appears. Suppress stderr and return `false` if the session doesn't exist (same approach as `exists()` for sessions).
2. `is_pane_dead()`: run `tmux_cmd().args(["-L", socket, "display-message", "-t", format!("{}:{}", session, window), "-p", "#{pane_dead}"])`. Parse stdout — `"1"` means dead, `"0"` means alive. Return error if window doesn't exist.
3. `pane_pid()`: similar to `is_pane_dead()` but queries `#{pane_pid}`. Parse to `u32`. Return error if window doesn't exist.
4. `list_windows()`: use a format string like `#{window_index}|#{window_name}|#{pane_pid}|#{pane_dead}` to get all fields in one call. Split each line by `|` and construct `TmuxWindow` structs. Handle empty output (no windows) gracefully. Note: the `|` delimiter is safe because the name validation regex (`[a-zA-Z0-9_-]` from Task 01) excludes `|` — this is an intentional coupling, not an accident.

## Acceptance Criteria

1. **Window Exists — True**
   - Given a session with a window named `bob`
   - When `window_exists("bob")` is called
   - Then it returns `true`

2. **Window Exists — False**
   - Given a session with no window named `bob`
   - When `window_exists("bob")` is called
   - Then it returns `false` (no error, no panic)

3. **Dead Pane Detection — Live Process**
   - Given a window running `sleep 300`
   - When `is_pane_dead("bob")` is called
   - Then it returns `Ok(false)`

4. **Dead Pane Detection — Exited Process**
   - Given a window whose command has exited (`true` command, with `remain-on-exit on`)
   - When `is_pane_dead("bob")` is polled after the process exits
   - Then it returns `Ok(true)`
   - Test approach: poll `is_pane_dead()` with a short retry loop (e.g., 5 attempts, 100ms apart) rather than a single check, to avoid flaky tests from tmux's async pane state update.

5. **PID Retrieval**
   - Given a window running `sleep 300`
   - When `pane_pid("bob")` is called
   - Then it returns `Ok(pid)` where pid matches a running process

6. **List Windows**
   - Given a session with windows `bob` and `cos`
   - When `list_windows()` is called
   - Then it returns a `Vec<TmuxWindow>` with entries for both, each containing index, name, pane_pid, and dead status

7. **Non-Existent Window Error**
   - Given a session with no window named `ghost`
   - When `pane_pid("ghost")` or `is_pane_dead("ghost")` is called
   - Then it returns an `Err` with a clear message including the window name
