# Task 03: Implement window removal, `session_info()`, `attach()`, and integration tests

## Context
- **Story**: #6 — STORY-02: Implement window lifecycle in `TmuxSession` — create, query, and manage windows
- **Requirements**: SESS-02, LIFE-03
- **Acceptance Criteria**: AC-04 (partial), AC-08 (partial)
- **Design Doc**: team/specs/botminter/1-tmux-agent-sessions/design.md

## Objective
Complete the window lifecycle API: `remove_window()`, `remove_dead_window()` (safe removal — only removes if dead), `session_info()` (aggregates session state for status display), and `attach()` (exec into the tmux session for operator access). Write comprehensive integration tests covering the full window lifecycle including failure paths.

## Background
Window removal is used during restart scenarios: when `bm start bob` is called and bob's window has a dead pane from a previous run, the dead window must be removed before creating a fresh one. `remove_dead_window()` provides a safe wrapper that checks `is_pane_dead()` first — it won't kill a live process's window.

`session_info()` aggregates the session state into a `SessionInfo` struct that STORY-04 will use for `bm status` display. `attach()` execs into the tmux session — this call blocks until the user detaches (Ctrl-b d). STORY-04 will wire `attach()` into `bm attach`.

## Technical Requirements
1. Implement `pub fn kill_window_process(&self, name: &str) -> Result<()>` — sends SIGTERM to the process inside the pane via `libc::kill(pid, SIGTERM)` using the PID from `pane_pid()`. The window stays alive due to `remain-on-exit on`. This is how `bm stop` terminates agents while preserving post-mortem scrollback.
2. Implement `pub fn remove_window(&self, name: &str) -> Result<()>` — runs `tmux kill-window -t <session>:<name>`
3. Implement `pub fn remove_dead_window(&self, name: &str) -> Result<()>` — checks `is_pane_dead(name)?` first, only removes if dead, no-op if window doesn't exist or pane is alive
3. Implement `pub fn session_info(&self) -> Result<SessionInfo>` — calls `list_windows()`, constructs `SessionInfo` with session_name, socket_name, windows list, and attach_command string (`tmux -L botminter attach-session -t <session>`)
4. Implement `pub fn attach(&self, window: Option<&str>) -> Result<()>` — execs `tmux -L botminter attach-session -t <session>` (no window target) or `tmux -L botminter attach-session -t <session>:<window>` (with window target). This call blocks until the user detaches.
5. All methods MUST use `tmux_cmd()` helper and `-L <socket>`
6. `attach()` MUST use `exec` semantics (replaces current process) or `status()` (blocks until detach) — NOT `output()` which would capture stdout

## Dependencies
- Task 01 must be complete (`create_window()`)
- Task 02 must be complete (query methods — `is_pane_dead()`, `list_windows()`)

## Implementation Approach
1. `remove_window()`: `tmux_cmd().args(["-L", socket, "kill-window", "-t", target])` where target is `"{session}:{name}"`. Check exit status.
2. `remove_dead_window()`: call `is_pane_dead(name)?`. If true, call `remove_window(name)`. If false (live process), return `Ok(())` silently. If window doesn't exist, return `Ok(())`.
3. `session_info()`: call `list_windows()`, construct `SessionInfo { session_name, socket_name, windows, attach_command: format!("tmux -L {} attach-session -t {}", socket, session) }`.
4. `attach()`: build args based on window parameter. Use `.status()` to block until detach. The calling code (STORY-04's `shell()`) must tolerate this blocking behavior.
5. Write integration tests covering the full lifecycle. Each test MUST use a unique team/session name (e.g., include a random suffix or test function name) to avoid collisions when `cargo test` runs tests in parallel:
   - Create session → create two windows (one long-running `sleep 300`, one immediate-exit `true`) → list shows both → detect dead pane on `true` window → `remove_dead_window()` removes only the dead one → live window untouched
   - `remove_dead_window()` on a live window is a no-op
   - `session_info()` returns correct data (session name, window count, attach command)
   - Failure paths: `remove_window()` on non-existent window returns error; `create_window()` when session doesn't exist returns clear error

## Acceptance Criteria

1. **Kill Window Process**
   - Given a window `bob` running `sleep 300`
   - When `kill_window_process("bob")` is called
   - Then the process receives SIGTERM, the window remains in the session, and `is_pane_dead("bob")` eventually returns true

2. **Remove Window**
   - Given a session with window `bob`
   - When `remove_window("bob")` is called
   - Then `window_exists("bob")` returns false

2. **Remove Dead Window — Dead Pane**
   - Given a window `bob` with a dead pane
   - When `remove_dead_window("bob")` is called
   - Then the window is removed

3. **Remove Dead Window — Live Pane (No-Op)**
   - Given a window `bob` with a live process (pid running)
   - When `remove_dead_window("bob")` is called
   - Then the window remains and the process is undisturbed

4. **Session Info**
   - Given a session with 2 windows
   - When `session_info()` is called
   - Then it returns `SessionInfo` with correct session_name, socket_name, 2 windows, and a valid attach_command string

5. **Attach — Basic**
   - Given a running session
   - When `attach(None)` is called and immediately detached
   - Then the method returns `Ok(())`
   - Test approach: spawn `attach()` in a background thread, poll for the client via `tmux -L botminter list-clients` until it appears, then send `tmux -L botminter send-keys -t <session> C-b d` to detach. Set a test timeout to prevent CI hangs.

6. **Attach — With Window Target**
   - Given a session with windows `bob` and `cos`
   - When `attach(Some("bob"))` is called
   - Then the tmux attach targets `<session>:bob`

7. **Full Lifecycle Integration**
   - Given tmux is installed
   - When the integration test runs: create session → add two windows (sleep + true) → list → detect dead → remove dead → verify live remains
   - Then each step produces the expected result

8. **Failure — Remove Non-Existent Window**
   - Given no window named `ghost` exists
   - When `remove_window("ghost")` is called
   - Then it returns an `Err`
