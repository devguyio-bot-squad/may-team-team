# Task 01: Implement `create_window()` with env var passing and PID retrieval

## Context
- **Story**: #6 — STORY-02: Implement window lifecycle in `TmuxSession` — create, query, and manage windows
- **Requirements**: SESS-02, SESS-03
- **Acceptance Criteria**: AC-04 (partial — pane output capture verified via integration test, not via `bm start` E2E)
- **Design Doc**: team/specs/botminter/1-tmux-agent-sessions/design.md

## Objective
Add `create_window()` to `TmuxSession` — the method that creates a named tmux window running a command with environment variables passed via `-e` flags. Returns the process PID inside the pane. This is the core primitive that STORY-03 will use to launch agents.

## Background
Each agent member runs in its own named tmux window within the team session. The window is created with `tmux new-window` using the `-e` flag to pass environment variables (credentials, bridge tokens) securely — they are passed through tmux's internal mechanism, not embedded in the shell command string. After creating the window, the method queries `#{pane_pid}` to get the actual process PID for state tracking, then validates the PID is alive via `kill(pid, 0)` to catch immediate failures (e.g., binary not found).

## Technical Requirements
1. Implement `pub fn create_window(&self, name: &str, cmd: &[&str], cwd: &Path, envs: &[(&str, &str)]) -> Result<u32>`
2. Validate window name against `[a-zA-Z0-9_-]` (same pattern as team name validation in `new()`)
3. Construct command: `tmux -L botminter new-window -t <session> -n <name> -c <cwd> -e K1=V1 -e K2=V2 -- <cmd> <args...>`
4. After window creation, query PID via `tmux -L botminter display-message -t <session>:<name> -p '#{pane_pid}'`
5. Validate PID is alive via `kill(pid, 0)` (libc or nix crate) — if the process already exited, return an error with exit status from `#{pane_dead_status}` rather than a stale PID
6. All tmux commands MUST use the `tmux_cmd()` helper (unsets `TMUX_TMPDIR`)
7. Env vars MUST be passed via `-e` flags, NOT shell string interpolation — secrets must not appear in `#{pane_start_command}` or `ps` output
8. Command arguments MUST be passed as a list after `--`, not as a shell string

## Dependencies
- STORY-01 must be complete (module structure, `TmuxSession` struct, `tmux_cmd()` helper, session lifecycle methods)
- A tmux session must exist before creating windows (caller responsibility — `create()` from STORY-01)

## Implementation Approach
1. Extract the name validation regex into a shared helper (or reuse from `TmuxSession::new()`) since both team names and window names use `[a-zA-Z0-9_-]`
2. Build the tmux command args dynamically: start with `["-L", socket, "new-window", "-t", session, "-n", name, "-c", cwd_str]`, append `-e K=V` for each env pair, append `--` then the command parts
3. Run the command and check exit status — return descriptive error on failure (include session name, window name, and stderr)
4. Query `#{pane_pid}` via a separate `display-message` command — tmux forks the pane process synchronously during `new-window`, so the PID is available immediately
5. Parse the PID string to `u32`
6. Validate PID: `unsafe { libc::kill(pid as i32, 0) }` — if it returns -1 with `ESRCH`, the process exited; query `#{pane_dead_status}` for the exit code and return an error
7. Return the validated PID

## Acceptance Criteria

1. **Window Creation with Command**
   - Given a tmux session exists
   - When `create_window("bob", &["sleep", "300"], workspace, &[])` is called
   - Then a window named `bob` exists in the session and `pane_pid` returns a valid, running PID

2. **Environment Variable Passing**
   - Given a tmux session exists
   - When `create_window("test", &["sleep", "300"], workspace, &[("SECRET", "token123")])` is called
   - Then the env var is accessible inside the window but NOT visible in `#{pane_start_command}` or `ps -p <pid> -o args=`

3. **Window Name Validation — Valid**
   - Given valid window names (`bob`, `agent_1`, `my-worker`)
   - When `create_window()` is called with each
   - Then it returns `Ok` with a valid PID

4. **Window Name Validation — Invalid**
   - Given invalid window names (`my;window`, `win:name`, `win name`, empty string)
   - When `create_window()` is called with each
   - Then it returns an `Err` describing the invalid characters

5. **Immediate Exit Detection — Non-Zero Exit**
   - Given a tmux session exists
   - When `create_window("fail", &["false"], workspace, &[])` is called (command exits immediately with status 1)
   - Then it returns an `Err` with context about the exit status, not a stale PID

6. **Immediate Exit Detection — Binary Not Found**
   - Given a tmux session exists
   - When `create_window("missing", &["__nonexistent_binary__"], workspace, &[])` is called
   - Then it returns an `Err` with a clear message (not a stale PID)

7. **No Session Error**
   - Given no tmux session exists
   - When `create_window()` is called
   - Then it returns an `Err` with a clear message (not a tmux stderr dump)
