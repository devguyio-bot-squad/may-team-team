## Objective
Implement `TmuxSession::check_tmux_available()` which runs `tmux -V`, parses the version string into a `TmuxVersion { major, minor }` struct, and returns an error if the version is below 3.0. This is the prerequisite check that STORY-03 will wire into `start_local_members()`.

## Background
tmux version strings have several known formats that must be handled:
- Standard: `tmux 3.4` (major.minor)
- With suffix: `tmux 3.3a` (letter suffix on minor)
- Release candidate: `tmux 3.2-rc` (dash suffix)
- Development: `tmux next-3.4` (prefix before version number)

The design mandates tmux 3.0+ because `remain-on-exit on` behavior and `-e` flag for env vars are stable from 3.0 onwards.

All `Command::new("tmux")` invocations in the tmux module MUST unset `TMUX_TMPDIR` via `.env_remove("TMUX_TMPDIR")` to enforce the secure default socket directory (`/tmp/tmux-<uid>/`). This is design decision D-09.

## Technical Requirements
1. Implement `pub fn check_tmux_available() -> Result<TmuxVersion>` as a static method on `TmuxSession`
2. Run `tmux -V` via `Command::new("tmux").arg("-V").env_remove("TMUX_TMPDIR")`
3. Parse the version string extracting the first two numeric components, ignoring suffixes (`a`, `-rc`, `next-`)
4. Return `TmuxVersion { major, minor }` on success
5. Return descriptive errors for: tmux not found in PATH, version string unparseable, version < 3.0
6. Establish the `TMUX_TMPDIR` unset pattern as a helper or convention for all tmux `Command` construction in the module

## Dependencies
- Task 01 must be complete (module structure, `TmuxVersion` struct defined)

## Implementation Approach
1. Complete the `TmuxVersion` struct in `mod.rs` (if only a skeleton from Task 01):
   - `pub major: u32`
   - `pub minor: u32`
   - Implement `Display` for user-facing messages (e.g., `"3.4"`)
2. Create a helper function for building tmux commands that always unsets `TMUX_TMPDIR`:
   ```rust
   fn tmux_cmd() -> Command {
       let mut cmd = Command::new("tmux");
       cmd.env_remove("TMUX_TMPDIR");
       cmd
   }
   ```
3. Implement `check_tmux_available()`:
   - Run `tmux_cmd().arg("-V").output()`
   - Handle the "not found" case (spawn failure) with an actionable error message
   - Parse stdout: extract numeric components using a regex or manual parsing
   - The version regex should handle: `tmux 3.4`, `tmux 3.3a`, `tmux next-3.4`, `tmux 3.2-rc`, `tmux 3.2-rc2`
   - Compare against minimum version (3, 0) and return error if below
4. Write unit tests for version parsing (pure function, no tmux needed):
   - Extract the parsing logic into a `parse_tmux_version(output: &str) -> Result<TmuxVersion>` function that can be unit tested independently
   - Test cases: `"tmux 3.3a"` → (3,3), `"tmux 3.4"` → (3,4), `"tmux next-3.4"` → (3,4), `"tmux 3.2-rc"` → (3,2), `"tmux 2.9a"` → error (below minimum), `"not tmux"` → error (unparseable)
5. Write integration test: `check_tmux_available()` returns `Ok` with a valid version on systems with tmux installed

## Acceptance Criteria

1. **Version Parsing — Standard Format**
   - Given the string `"tmux 3.4"`
   - When `parse_tmux_version()` is called
   - Then it returns `TmuxVersion { major: 3, minor: 4 }`

2. **Version Parsing — Letter Suffix**
   - Given the string `"tmux 3.3a"`
   - When `parse_tmux_version()` is called
   - Then it returns `TmuxVersion { major: 3, minor: 3 }`

3. **Version Parsing — Development Prefix**
   - Given the string `"tmux next-3.4"`
   - When `parse_tmux_version()` is called
   - Then it returns `TmuxVersion { major: 3, minor: 4 }`

4. **Version Parsing — Release Candidate**
   - Given the string `"tmux 3.2-rc"`
   - When `parse_tmux_version()` is called
   - Then it returns `TmuxVersion { major: 3, minor: 2 }`

5. **Minimum Version Enforcement**
   - Given `check_tmux_available()` detects tmux version 2.9
   - When the check runs
   - Then it returns an error stating "tmux 3.0+ is required (found 2.9)"

6. **tmux Not Found**
   - Given `tmux` is not in PATH
   - When `check_tmux_available()` is called
   - Then it returns an error naming "tmux" as the missing dependency

7. **TMUX_TMPDIR Unset**
   - Given `TMUX_TMPDIR` is set in the process environment
   - When any tmux command is constructed via the module's command helper
   - Then `TMUX_TMPDIR` is removed from the child process environment

8. **Integration — Real tmux**
   - Given tmux is installed on the system
   - When `check_tmux_available()` is called
   - Then it returns `Ok(TmuxVersion)` with `major >= 3`
