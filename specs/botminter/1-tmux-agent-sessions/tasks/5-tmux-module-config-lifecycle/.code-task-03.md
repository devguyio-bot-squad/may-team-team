# Task 03: Implement session lifecycle methods with integration tests

## Context
- **Story**: #5 — STORY-01: Add `formation/local/tmux/` module with config management and session lifecycle
- **Requirements**: TMUX-01, TMUX-04
- **Acceptance Criteria**: AC-02 (partial — session-level, not yet wired into `bm start`), AC-17
- **Design Doc**: team/specs/botminter/1-tmux-agent-sessions/design.md

## Objective
Implement the session lifecycle methods on `TmuxSession`: `exists()`, `create()`, `destroy()`, and `destroy_if_exists()`. These methods manage a tmux session on the dedicated `botminter` socket, using the branded config file. After this task, the complete STORY-01 deliverable is functional — tmux sessions can be created and destroyed programmatically with all the foundational infrastructure in place.

## Background
A tmux session is the top-level container. Each team gets one session (`bm-<team>`), and STORY-02 will add windows within it (one per agent). The session is created on a dedicated socket (`-L botminter`) to isolate BotMinter from the operator's personal tmux.

The `create()` method must load the custom tmux.conf via `-f` to apply the branded configuration (status bar, `remain-on-exit on`, scrollback, etc.). `TmuxConfig::ensure_written()` (from Task 01) must be called before creating a session to guarantee the config file exists on disk.

## Technical Requirements
1. Implement `pub fn exists(&self) -> bool` — runs `tmux -L botminter has-session -t bm-<team>` and returns true if exit code is 0
2. Implement `pub fn create(&self) -> Result<()>` — runs `tmux -L botminter new-session -d -s bm-<team> -f <config_path>` where `config_path` is from `TmuxConfig::path()`
3. Implement `pub fn destroy(&self) -> Result<()>` — runs `tmux -L botminter kill-session -t bm-<team>`
4. Implement `pub fn destroy_if_exists(&self) -> Result<()>` — calls `exists()`, then `destroy()` if true, no-op otherwise
5. All commands MUST use the `tmux_cmd()` helper from Task 02 (ensures `TMUX_TMPDIR` is unset)
6. All commands MUST use `-L {socket_name}` for socket isolation
7. `create()` MUST call `TmuxConfig::ensure_written()?` before invoking tmux to guarantee the config file exists
8. Error messages MUST include the session name and socket name for debuggability

## Dependencies
- Task 01 must be complete (module structure, `TmuxConfig`, `TmuxSession::new()`)
- Task 02 must be complete (`tmux_cmd()` helper, `check_tmux_available()`)
- tmux 3.0+ must be installed for integration tests

## Implementation Approach
1. Implement `exists()`:
   - `tmux_cmd().args(["-L", &self.socket_name, "has-session", "-t", &self.session_name]).status()`
   - Return `status.success()` — `has-session` exits 0 if the session exists, 1 if not
   - Suppress stderr (tmux prints "no server running" to stderr when no socket exists — this is not an error for `exists()`)
2. Implement `create()`:
   - Call `TmuxConfig::ensure_written()?` first
   - `tmux_cmd().args(["-L", &self.socket_name, "new-session", "-d", "-s", &self.session_name, "-f", config_path.to_str()])`
   - Check exit status and return descriptive error on failure
3. Implement `destroy()`:
   - `tmux_cmd().args(["-L", &self.socket_name, "kill-session", "-t", &self.session_name])`
   - Check exit status — error if session doesn't exist (caller should use `destroy_if_exists()` for idempotent cleanup)
4. Implement `destroy_if_exists()`:
   - `if self.exists() { self.destroy()? }; Ok(())`
5. Write integration tests (require real tmux):
   - **Lifecycle test**: `create()` → `exists()` returns true → `destroy()` → `exists()` returns false
   - **Idempotent destroy**: `destroy_if_exists()` on non-existent session → no error
   - **Double create**: `create()` twice → second call returns error (session already exists)
   - **Config file verification**: after `create()`, verify `~/.config/botminter/tmux.conf` exists with `0600` permissions and expected content
   - **Socket isolation**: after `create()`, verify session is on the `botminter` socket via `tmux -L botminter list-sessions`
6. Add CI prerequisite note: integration tests require `apt install -y tmux` in CI setup

## Acceptance Criteria

1. **Session Creation**
   - Given no tmux session exists for the team
   - When `TmuxSession::create()` is called
   - Then a tmux session named `bm-<team>` exists on the `botminter` socket

2. **Session Existence Check — Exists**
   - Given a tmux session `bm-<team>` was created
   - When `TmuxSession::exists()` is called
   - Then it returns `true`

3. **Session Existence Check — Not Exists**
   - Given no tmux session exists for the team
   - When `TmuxSession::exists()` is called
   - Then it returns `false` (no error, no panic)

4. **Session Destruction**
   - Given a tmux session `bm-<team>` exists
   - When `TmuxSession::destroy()` is called
   - Then `exists()` returns `false`

5. **Idempotent Destroy**
   - Given no tmux session exists for the team
   - When `TmuxSession::destroy_if_exists()` is called
   - Then no error is returned

6. **Config File Written Before Session**
   - Given `TmuxSession::create()` is called
   - When the session is created
   - Then `~/.config/botminter/tmux.conf` exists on disk with `0600` permissions

7. **Socket Isolation**
   - Given a tmux session was created via `TmuxSession::create()`
   - When `tmux -L botminter list-sessions` is run
   - Then the session `bm-<team>` appears in the output
   - And `tmux list-sessions` (default socket) does NOT show `bm-<team>`

8. **Integration Test — Full Lifecycle**
   - Given tmux is installed
   - When the integration test runs create → exists → destroy → exists
   - Then each step produces the expected result with no test flakiness
