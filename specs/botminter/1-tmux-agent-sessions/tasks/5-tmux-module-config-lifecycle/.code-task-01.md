# Task 01: Create tmux module structure with TmuxConfig and TmuxSession::new()

## Context
- **Story**: #5 — STORY-01: Add `formation/local/tmux/` module with config management and session lifecycle
- **Requirements**: TMUX-04, BRAND-01
- **Acceptance Criteria**: AC-17
- **Design Doc**: team/specs/botminter/1-tmux-agent-sessions/design.md

## Objective
Create the `formation/local/tmux/` directory module with two files: `mod.rs` (public API — `TmuxSession` struct and constructor) and `config.rs` (`TmuxConfig` — config file management). Register the module in `formation/local/mod.rs`. After this task, the codebase can construct `TmuxSession` handles with validated team names and write the branded tmux.conf to the user config home.

## Background
The BotMinter local formation currently launches agent processes as invisible background processes with stdio redirected to `/dev/null`. This epic adds tmux-based terminal multiplexing. This task creates the foundational module that all subsequent tmux work builds on.

The codebase has two config directories:
- `~/.config/botminter/` — user config home (profiles, minty config, VM templates). Resolved via `dirs::config_dir().join("botminter")`.
- `~/.botminter/` — agent runtime state (`config.yml`, `state.json`, credentials). Resolved via `dirs::home_dir().join(".botminter")` in `config::config_dir()`.

The tmux.conf is user configuration, so it belongs in `~/.config/botminter/tmux.conf`.

## Technical Requirements
1. Create `crates/bm/src/formation/local/tmux/mod.rs` with `TmuxSession` struct and public API
2. Create `crates/bm/src/formation/local/tmux/config.rs` with `TmuxConfig` struct
3. The tmux.conf content MUST be stored as a file in the project repo (e.g., `crates/bm/src/formation/local/tmux/tmux.conf`) and loaded via `include_str!()` — NOT as a `const &str` inline in Rust source
4. `TmuxConfig::path()` MUST resolve to `~/.config/botminter/tmux.conf` using `dirs::config_dir().join("botminter").join("tmux.conf")`, matching the existing profile/minty pattern
5. `TmuxConfig::ensure_written()` MUST use atomic write (temp file + rename) with `0600` permissions, following the pattern in `bridge/manifest.rs:172-194`
6. `TmuxSession::new(team_name)` MUST validate team name against `[a-zA-Z0-9_-]` regex, construct `socket_name = "botminter"` and `session_name = "bm-{team}"`
7. Register `mod tmux;` in `formation/local/mod.rs`
8. Per ADR-0006: new modules are directory modules. Per ADR-0003/D-03: tmux is a formation concern under `formation/local/`

## Dependencies
- `dirs` crate (already in dependencies — used by `profile::profiles_dir()`)
- `tempfile` crate (already in dev-dependencies — used for test setup)
- No new crate dependencies required

## Implementation Approach
1. Create the tmux.conf file at `crates/bm/src/formation/local/tmux/tmux.conf` with the branded config content from the design doc (status bar with "botminter", keybinding hints, `remain-on-exit on`, 50k scrollback, mouse on, `update-environment ""`)
2. Create `config.rs` with `TmuxConfig` struct:
   - `pub fn config_content() -> &'static str` — returns `include_str!("tmux.conf")`
   - `pub fn path() -> Result<PathBuf>` — resolves `dirs::config_dir()?.join("botminter").join("tmux.conf")`
   - `pub fn ensure_written() -> Result<()>` — creates parent dir if needed, atomic write (temp file in same dir + rename), sets `0600` permissions before rename
3. Create `mod.rs` with `TmuxSession` struct:
   - Fields: `socket_name: String`, `session_name: String`, `config_path: PathBuf`
   - `pub fn new(team_name: &str) -> Result<Self>` — validates name, derives socket/session names, resolves config path via `TmuxConfig::path()`
   - Define `TmuxVersion`, `TmuxWindow`, `SessionInfo` structs (public types, can be empty/minimal — completed in tasks 02 and 03)
4. Add `mod tmux;` to `formation/local/mod.rs`

## Acceptance Criteria

1. **Module Structure**
   - Given the codebase has no `formation/local/tmux/` directory
   - When this task is complete
   - Then `formation/local/tmux/mod.rs` and `formation/local/tmux/config.rs` exist, compile, and are registered in `formation/local/mod.rs`

2. **Name Validation — Valid Names**
   - Given valid team names (`my-team`, `team_1`, `MyTeam123`)
   - When `TmuxSession::new()` is called with each
   - Then it returns `Ok` with `session_name = "bm-{team}"` and `socket_name = "botminter"`

3. **Name Validation — Invalid Names**
   - Given invalid team names (`my;team`, `team:name`, `team name`, `team/path`, empty string)
   - When `TmuxSession::new()` is called with each
   - Then it returns an `Err` with a message describing the invalid characters

4. **Config Content**
   - Given `TmuxConfig::config_content()` is called
   - When the returned content is inspected
   - Then it is non-empty, contains `remain-on-exit on`, contains `botminter` branding, and contains keybinding hints (`C-b n:next`, `C-b [:scroll`)

5. **Config Path**
   - Given `TmuxConfig::path()` is called
   - When the returned path is inspected
   - Then it ends with `.config/botminter/tmux.conf`

6. **Config File Write**
   - Given `TmuxConfig::ensure_written()` is called
   - When the config file is inspected on disk
   - Then the file exists at `~/.config/botminter/tmux.conf`, has `0600` permissions, and contains the expected config content

7. **Atomic Write Safety**
   - Given `TmuxConfig::ensure_written()` is called
   - When the operation completes
   - Then no `.tmp` file remains in the config directory (temp file was renamed, not left behind)
