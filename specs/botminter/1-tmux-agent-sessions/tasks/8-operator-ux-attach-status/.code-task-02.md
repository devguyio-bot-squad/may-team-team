# Task 02: Implement `bm attach` with tmux session attachment

## Context
- **Story**: [#8 — STORY-04: Add operator UX — bm attach, bm status tmux info, prerequisites, E2E tests](https://github.com/devguyio-bot-squad/may-team-team/issues/8)
- **Requirements**: UX-01
- **Acceptance Criteria**: AC-10, AC-10b, AC-14, AC-19
- **Design Doc**: team/specs/botminter/1-tmux-agent-sessions/design.md

## Objective
Make `bm attach [member]` attach to the tmux session, with optional per-member window targeting, a pre-attach cheat sheet, and nested-session detection. This involves a breaking trait change to `Formation::shell()`.

## Background
`Formation::shell()` currently has signature `fn shell(&self) -> Result<()>` and the `LinuxLocalFormation` implementation returns an error: "shell() is not applicable for local formation." The CLI's `Attach` command variant (in `cli.rs`, line 355-360) has only a `team` field, no `member` field. `commands/attach.rs` calls `local_formation.shell()` with no arguments at line 22.

This task changes `shell()` to accept an optional member name, updates all trait implementors, wires the member argument through the CLI and command handler, and implements the actual tmux attach logic with UX touches (cheat sheet, nested session warning).

## Technical Requirements
1. Change `Formation::shell()` trait method signature to `fn shell(&self, member: Option<String>) -> Result<()>`
2. Update ALL trait implementors:
   - `LinuxLocalFormation::shell(member)` — implement tmux attach (see below)
   - `MacosLocalFormation::shell(_member)` — keep existing bail, accept and ignore parameter
   - `MockFormation::shell(_member)` — keep existing `Ok(())`, accept and ignore parameter
3. Add optional positional `member` argument to the `Attach` CLI variant in `cli.rs`
4. Update `commands/attach.rs` `run()` to accept and pass the member argument through to `Formation::shell(member)`
5. `LinuxLocalFormation::shell(member)` implementation:
   - Construct `TmuxSession::new(&self.team_name)?`
   - If `!tmux.exists()`, bail with: "No tmux session found. Start members first with: bm start"
   - Check `$TMUX` env var — if set, print warning to stderr about nested sessions with detach advice, then proceed
   - Print tmux cheat sheet to stderr before attaching:
     ```
     Ctrl-b n     next window
     Ctrl-b p     prev window
     Ctrl-b [     scroll mode (q to exit)
     Ctrl-b d     detach (return to shell)
     ```
   - Call `tmux.attach(member.as_deref())` — `None` for full session, `Some("bob")` for direct window
6. The `attach()` call blocks until the user detaches — this is correct for `shell()` semantics

## Dependencies
- `TmuxSession` struct with `new()`, `exists()`, and `attach()` methods from STORY-01/02 (must be implemented)
- Task 01 of this story (prerequisites check) should be done first so `bm start` validates tmux before creating sessions

## Implementation Approach
1. **Trait change**: Update `Formation::shell()` signature in `formation/mod.rs` to accept `member: Option<String>`
2. **Update implementors**: Fix compile errors in all three implementors — `LinuxLocalFormation`, `MacosLocalFormation`, `MockFormation`
3. **CLI change**: Add `member: Option<String>` as an optional positional argument to the `Attach` variant in `cli.rs`
4. **Command handler**: Update `commands/attach.rs` `run()` to accept the member parameter and pass it to `formation.shell(member)`
5. **Implement attach logic**: In `LinuxLocalFormation::shell()`, replace the bail with tmux session attachment including cheat sheet, nested warning, and error handling
6. **Integration tests**: Test attach with no session (error), attach with session (success + immediate detach via `send-keys`), attach with specific member window, and nested session warning detection

## Acceptance Criteria

1. **Full session attach**
   - Given agents are running in tmux
   - When the operator runs `bm attach`
   - Then the operator is attached to the `bm-<team>` tmux session

2. **Per-member attach**
   - Given agents are running in tmux
   - When the operator runs `bm attach bob`
   - Then the operator is attached to the `bm-<team>` session with window `bob` selected

3. **No session error**
   - Given no tmux session exists
   - When the operator runs `bm attach`
   - Then the command exits with error message "No tmux session found. Start members first with: bm start"

4. **Nested session warning**
   - Given the operator is already inside a tmux session (`$TMUX` is set)
   - When the operator runs `bm attach`
   - Then stderr contains a warning mentioning "already inside a tmux session", explains BotMinter uses a separate server (`-L botminter`), advises detaching first with `Ctrl-b d`, and the attach proceeds anyway

5. **Cheat sheet display**
   - Given a tmux session exists
   - When the operator runs `bm attach`
   - Then stderr contains the lines `Ctrl-b n     next window`, `Ctrl-b p     prev window`, `Ctrl-b [     scroll mode (q to exit)`, and `Ctrl-b d     detach (return to shell)` before the tmux session is entered

6. **Non-existent member window**
   - Given agents are running in tmux but no window named `alice` exists
   - When the operator runs `bm attach alice`
   - Then the command exits with an error listing the available member windows (e.g., "Window 'alice' not found. Available: bob, cos, sentinel")

7. **Trait compatibility**
   - Given the `Formation::shell()` trait signature changed
   - When the project is compiled
   - Then all trait implementors (`LinuxLocalFormation`, `MacosLocalFormation`, `MockFormation`) compile without errors, `MacosLocalFormation::shell(None)` still returns an error, and `MockFormation::shell(None)` still returns `Ok(())`
