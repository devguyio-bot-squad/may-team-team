**Exploratory tests** are in `crates/bm/tests/exploratory/` with phase scripts (`phase-b.sh` through `phase-h.sh`). These need tmux verification added to relevant phases.

## Technical Requirements

### Branded config verification
1. Verify the status bar contains "botminter" text
2. Verify the status bar shows the session name (`bm-<team>`)
3. Verify the status bar shows window tabs with member names
4. Verify keybinding hints are visible: `C-b n:next`, `C-b p:prev`, `C-b [:scroll`, `C-b d:detach`
5. Verification approach: create a tmux session with the config, use `tmux display-message` or `tmux show-options` to verify config directives are loaded

### E2E test updates
1. Update `attach_local_formation` test in `scenarios/operator_journey.rs` — change from asserting failure to asserting successful tmux attachment (attach + immediate detach via `send-keys C-b d`)
2. After `bm start`: verify `tmux -L botminter has-session -t bm-<team>` succeeds
3. After `bm start`: verify `tmux -L botminter list-windows` returns expected window names
4. `bm status` output: verify it contains tmux session name, "bm attach", and raw tmux command
5. After `bm stop`: verify windows still exist with dead panes

### Exploratory test updates
1. Add tmux verification to relevant phase scripts:
   - Session creation after `bm start`
   - Window naming matches member names
   - Post-mortem scrollback inspection after `bm stop`
   - Cheat sheet output on `bm attach`

## Dependencies
- Tasks 01, 02, and 03 of this story must be complete (prerequisites, attach, and status features)
- STORY-01/02/03 tmux infrastructure must be implemented
- tmux must be installed in the CI environment

## Implementation Approach
1. **Branded config integration test**: Write a test that creates a tmux session using the BotMinter config, then queries `tmux show-options` and `tmux display-message -p` to verify status bar format string contains expected elements ("botminter", keybinding hints)
2. **Update `attach_local_formation` E2E test**: Change the assertion from expecting an error to expecting success. Start a tmux session, run `bm attach`, immediately send `C-b d` to detach, assert exit code 0
3. **Add E2E tmux assertions**: In the operator journey test, after `bm start`, add assertions for `tmux -L botminter has-session` and `tmux -L botminter list-windows` output. After `bm stop`, verify `tmux -L botminter list-windows` still shows windows
4. **Add E2E status assertion**: After `bm start`, run `bm status` and assert output contains tmux session name and attach commands
5. **Update exploratory test phases**: Add tmux checks to the relevant phase scripts using `tmux -L botminter` commands

## Acceptance Criteria

1. **Branded status bar**
   - Given agents are running in tmux with the BotMinter config
   - When `tmux -L botminter show-options -g status-left` is queried
   - Then the output contains the string "botminter"
   - And `tmux -L botminter list-windows -t bm-<team> -F '#{window_name}'` returns member names

2. **Keybinding hints visible**
   - Given the custom tmux.conf is loaded
   - When `tmux -L botminter show-options -g status-right` is queried
   - Then the output contains `C-b n:next`, `C-b p:prev`, `C-b [:scroll`, and `C-b d:detach`

3. **E2E: tmux session exists after start**
   - Given the E2E test environment
   - When `bm start` completes
   - Then `tmux -L botminter has-session -t bm-<team>` succeeds and `list-windows` returns expected member windows

4. **E2E: attach succeeds**
   - Given agents are running
   - When `bm attach` is called in the E2E test
   - Then it attaches successfully (verified by immediate detach without error)

5. **E2E: status shows tmux info**
   - Given agents are running
   - When `bm status` is called
   - Then output contains tmux session name, window count, and both attach commands

6. **E2E: post-stop windows preserved**
   - Given agents were stopped via `bm stop`
   - When `tmux -L botminter list-windows` is queried
   - Then windows still exist (dead panes from `remain-on-exit on`)

7. **E2E: raw attach command is functional**
   - Given `bm status` output contains a `tmux -L botminter attach -t bm-<team>` command
   - When that exact command is executed
   - Then it attaches to the session (verified by immediate detach via `send-keys C-b d` without error)

8. **Exploratory tests cover tmux**
   - Given the exploratory test phase scripts in `crates/bm/tests/exploratory/`
   - When relevant phases run
   - Then the scripts include: (a) `tmux -L botminter has-session` assertion after `bm start`, (b) `tmux -L botminter list-windows` output matches member names, (c) `tmux -L botminter capture-pane` retrieves scrollback after `bm stop`, (d) `bm attach` stderr contains cheat sheet text
